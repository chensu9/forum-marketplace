# 🌐 NEXUS BOARD // Matrix Syndicate Terminal

**Nexus Board** — это децентрализованный киберпанк-форум и теневой маркетплейс, стилизованный под хакерский терминал. Платформа объединяет в себе функционал социальной сети, защищенного мессенджера и торговой площадки с системой безопасных сделок (Escrow).

## ⚡ Особенности системы (Features)

* **Строгая иерархия (Роли):** Система уровней доступа (`USER`, `MODERATOR`, `ADMIN`) с глобальными бейджами.
* **Терминальный интерфейс:** Полностью кастомный UI в стиле командной строки (зеленый неон, ASCII-рамки, глитч-эффекты).
* **Secure Comms (Мессенджер):** Встроенная система P2P-сообщений в формате логов с глобальным лайв-счетчиком непрочитанных писем (`INBOX`).
* **Escrow Market (Гарант):** Безопасные сделки. Заморозка средств при покупке (`IN_PROGRESS`) и перевод продавцу только после подтверждения получения (`COMPLETED`).
* **Система модерации:** Пользовательские репорты на нарушения и суровый Бан-хаммер (Red Screen of Death) для нарушителей.
* **Кастомизация узла:** Пользователи могут менять описание профиля и загружать кастомные фоновые изображения.

## 🛠 Технический стек

* **Фреймворк:** [Next.js 14+ (App Router)](https://nextjs.org/)
* **База данных:** PostgreSQL + [Prisma ORM](https://www.prisma.io/)
* **Стилизация:** [Tailwind CSS](https://tailwindcss.com/)
* **Аутентификация:** [Auth.js (NextAuth v5)](https://authjs.dev/)

## 💻 ИНСТРУКЦИЯ ПО ЛОКАЛЬНОЙ УСТАНОВКЕ (Local Setup Guide)

Следуй этим шагам, чтобы развернуть собственный узел сети Nexus на своем компьютере.

### Шаг 0: Подготовка (Prerequisites)
Перед началом убедись, что на твоем ПК установлены:
1. **Node.js** (версия 18.17 или выше). [Скачать можно здесь](https://nodejs.org/).
2. **Git** для клонирования репозитория.
3. **PostgreSQL база данных**. Ты можешь установить её локально, либо за 1 минуту создать бесплатную облачную БД на [Supabase](https://supabase.com/) или [Neon.tech](https://neon.tech/).

### Шаг 1: Клонирование протокола
Открой терминал и клонируй репозиторий к себе на компьютер:
```bash
git clone [https://github.com/ТВОЙ_НИКНЕЙМ/nexusboard.git](https://github.com/ТВОЙ_НИКНЕЙМ/nexusboard.git)
cd nexusboard
(Не забудь заменить ссылку на свою, если репозиторий загружен на твой GitHub).

Шаг 2: Установка зависимостей
Установи все необходимые NPM-пакеты для работы системы:

Bash
npm install
Шаг 3: Настройка переменных окружения (Environment Variables)
Создай в корне проекта файл с названием .env. Вставь в него следующие строки:

Фрагмент кода
# Строка подключения к твоей базе данных PostgreSQL
DATABASE_URL="postgresql://user:password@localhost:5432/nexusboard"

# Секретный ключ для шифрования сессий Auth.js
AUTH_SECRET="сгенерируй_любой_случайный_длинный_набор_символов"
Где взять AUTH_SECRET?
Просто открой терминал и введи команду npx auth secret. Она сгенерирует надежный ключ и сама добавит его в твой файл .env. Либо придумай его сам (минимум 32 символа).

Шаг 4: Инициализация Базы Данных (Взлом Матрицы)
Теперь нужно создать все таблицы (пользователей, посты, сообщения) в твоей базе данных. Выполни эти две команды строго по очереди:

Bash
# 1. Отправляем схему в базу данных
npx prisma db push

# 2. Генерируем типы TypeScript для редактора
npx prisma generate
Шаг 5: Запуск сервера
Запускаем двигатели! Подними локальный сервер для разработки:

Bash
npm run dev
Если всё прошло успешно, терминал сообщит, что сервер запущен. Открой в браузере: http://localhost:3000

🔴 СЕКРЕТНЫЙ ШАГ: Как выдать себе права [ SYS_ADMIN ]
По умолчанию все новые пользователи регистрируются с правами USER. Чтобы получить доступ к командному центру (/admin) и бан-хаммеру, тебе нужно "взломать" базу данных и выдать себе высшие привилегии.

Зайди на свой локальный сайт (localhost:3000) и зарегистрируй новый аккаунт.

Не останавливая сервер, открой новую вкладку терминала в папке проекта.

Введи команду для запуска интерфейса базы данных:

Bash
npx prisma studio
В браузере откроется панель Prisma Studio (обычно на localhost:5555).

Кликни на таблицу User.

Найди в списке свой аккаунт, дважды кликни на колонку role и измени значение с USER на ADMIN.

Нажми зеленую кнопку Save 1 change в верхнем меню.

Закрой Prisma Studio (в терминале нажми Ctrl + C).

Готово! Возвращайся на форум, обнови страницу, и твой никнейм украсит красный бейдж [ SYS_ADMIN ]. Доступ к панели COMMAND_CENTER открыт.
